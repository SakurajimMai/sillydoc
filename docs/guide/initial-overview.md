## API 连接

然后是第二个按钮，长得像个插头，这是 API 连接设置，因为长得像插头就叫插头了。当有人说“看看插头”的时候，意思是让你点开这个然后截图发出来。

![](https://s2.ixacg.com/2026/04/09/1775740472.avif)

这里默认是 AI Horde，推荐谷歌 Gemini，A 社 Claude 等在线的聊天补全 API，所以要点 API 的下拉选单，换成聊天补全。注意这里是聊天补全，不是文本补全。下面的提示词后处理选择“合并相同角色连续的发言”。

![](https://s2.ixacg.com/2026/04/09/1775740578.avif)

注意有些预设或插件会特别要求提示词后处理选择别的，这时候选它们要求的那个就行。  
改完插头，再回去看预设，它已经变成“聊天补全预设”了，熟悉的样子。不过这个自带的 default 预设根本用不了，在实际和 AI 聊天之前，记得先去社区里找个预设用。

绿色锁链按钮是将预设和插头配置绑定，实际并不方便还容易捣乱，建议点成灰色的，这样就不绑定了。

### 触发策略

触发策略分三个：`蓝灯🔵`，`绿灯🟢`，`向量化⛓️`。酒馆自带的向量化 RAG 功能并不好用，也没人用，这里不做介绍。

- `蓝灯🔵` 一直触发，只要这本世界书开着，在全局或者被绑定到当前角色卡，这个条目的开关开着，这个条目的内容就会一直发给 AI。
- `绿灯🟢` 只在最近的正文中出现“绿灯关键词”时才触发，绿灯关键词就是上面图里的 `主要关键字`。而“最近的正文”的“最近”范围，是在 `全局世界信息/知识书激活设置` 里面设置的 `扫描深度`。如果你按推荐设置改成了 2，那这个范围就是最近 2 条消息，即最后一条 user 消息和最后一条 AI 消息。如果这两条消息里出现了绿灯关键词，绿灯条目的内容就会发给 AI。

注意绿灯关键词必须用英文逗号 `,` 分隔，不能用空格、中文逗号、分号等。我更推荐你点一下 `主要关键字` 最右边那个图标，这样敲一个关键词就按一下回车自动分隔，比较方便。

我们以一个绿灯角色条目举例。理想状况下，当剧情进展到和某角色相关，user 或 AI 的发言自然而然地提起对应角色的名字、特征、历史事件、相关物品等，从而触发绿灯。从这里我们可以联想到绿灯的两个问题。

如果剧情上要写某角色相关事情了，但上文刚好并未提到关键词；或者“正是 user 的这次发言把该角色引入了剧情”，又没有直接说绿灯关键词，就会导致 AI 在写相关事情的时候缺少资料，开始已读乱回。  
引入了相关角色信息后，AI 一直写这个角色相关的内容，每一次回复里都有关键词，越写上下文里有关内容就越多，有关内容越多就越想写，导致剧情无法正常推进。

可以看到这两个问题在某种程度上是矛盾的，本质上是因为绿灯靠正文中出现关键词激活这种简单粗暴的激活方式。而且，因为这种触发方式，绿灯也很难用来制作“分阶段人设”等，不靠具体关键词，而是靠角色“内在变化”，也就是上下文影响导致 AI 倾向改变的内容。

因此，现在很多角色卡都放弃了绿灯，转而用变量系统实现更精细的提示词控制。

### 位置、顺序、深度

位置，指的是世界书条目会被放在整段提示词里的位置。酒馆提供了 `角色定义前/后`，`作者注释前/后`，`示例消息前/后`，`D⚙️/👤/🤖` 共 7 个选项，但作者注释前后和示例消息前后在最近的角色卡中都几乎不用了，只介绍角色定义前后和 `D⚙️/👤/🤖`。

- 角色定义之前，对应预设中的 `World Info (before)`
- 角色定义之后，对应预设中的 `World Info (after)`
- `D⚙️/👤/🤖` 作为一条 role 为系统、user 或 AI 的消息，插入到聊天记录中，即预设里的 `Chat History`

在位置相同的前提下，条目之间的前后顺序由顺序决定，顺序越大越靠后。可以把顺序理解成第几个。

选择深度的时候，还会多出一个叫 `深度` 的选项，是用来控制这个条目在聊天记录里的相对位置的。

深度在酒馆里是一个很常用的概念。一句话概括，从聊天记录最底部往上数，第 N 条消息与第 N+1 条消息之间的位置，就是深度 N。画图表示的话就是这样：

![](https://s2.ixacg.com/2026/04/09/1775742371.avif)

从下向上数，第 0 条消息与第 1 条消息之间的“深度 0”，也就是 D0，是最后一条消息的下方，也是整个聊天记录的最底部；而如果把这个数字设置得特别大，比如 D999，条目内容就会位于整个聊天记录最上方，比第一条消息还靠上。

## 扩展程序

第六个是扩展，以后导入正则、装插件、调插件都在这里。

![](https://s2.ixacg.com/2026/04/09/1775740879.avif)

[酒馆助手](https://github.com/N0VI028/JS-Slash-Runner)，[小白X](https://github.com/RT15548/LittleWhiteBox)，[提示词模板](https://codeberg.org/zonde306/ST-Prompt-Template)，以上三个是很多角色卡及预设的依赖，都装一下。

![](https://s2.ixacg.com/2026/04/09/1775741589.avif)

注意酒馆助手和小白 X 的渲染器同一时间只能开一个，两个插件最好也同一时间只开一个，防冲突。酒馆助手里面也能安装 `酒馆助手脚本`，提供各种功能，具体方法是打开扩展页面的 `脚本库` 然后点导入或者安装的按钮，此外点那个内置库能安装几个比较常用的酒馆助手脚本。

小白 X 的方法也类似。注意，角色卡自带的脚本不需要手动再导入一次，会随角色卡一起导入到脚本库里。
